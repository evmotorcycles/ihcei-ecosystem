01 The problem

Your health signals measure activity. Collapse is a failure of follow-through.

Every operational health metric you have — last deploy, commit frequency, green CI, ticket volume — measures that something is moving. None of them measure whether the organization still closes the risks it raises against itself. That gap is where collapse hides.

Failure mode A · Zombie

Cosmetic activity masks a dead core

Automated dependency bumps and doc edits keep the "active" light green while the maintainer base has left. Backlog of self-flagged risk grows unbounded. The metadata lies.

Failure mode B · Bulk-close

End-of-life mass-closes hide the latency

A dying system mass-closes its trailing tickets at sunset, so naive latency looks great right as it dies. Any lagging metric is blind to it.

Both are everyday realities in large socio-technical systems. A useful collapse sensor has to survive them — not be fooled by them. τ_v was built to be.

02 The sensor

One measurement. Timestamps you already store.

τv = meani ∈ self-flagged risks ( tclosed(i) − traised(i) )   // capped, per unit

Enforcement latency: the time a system takes to resolve a risk it itself raised. Rising τ_v is loss of enforcement capacity — the leading edge of collapse — and it is computable from data every organization already keeps.

It is MTTR's honest cousin — but a leading indicator, not a post-mortem average, and scoped to self-raised risk so it can't be gamed by volume.

"Failed" repos looked faster (p = 0.75). Not a sensor failure — the label is contaminated: zombie projects mislabeled "survived" carry the highest τ_v, and archived repos were bulk-closed at sunset. Exactly the two failure modes from slide 01.

B · τ_v was right all along

Grade by actual maintenance status — the decay τ_v is built to sense — and the signal returns hard: dormant 73 d vs active 8 d (9×, p = 0.013; 11× and p = 0.006 once the bulk-close artifacts are removed).

The takeaway an SRE will recognize: τ_v didn't fail the messy telemetry — it pierced it. It flags the walking-dead systems that a last-commit heuristic waves through. It leads the lifecycle label instead of lagging it.
